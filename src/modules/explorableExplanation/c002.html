!!RED ALERT!!

Batten down the hatches! Everybody panic!

This room looks... oppressive. But this is the reality of the world the little dude lives in. The red tiles surround him. The red tiles envelop him. The red tiles are slowly closing in, with every waking breath.

The red tiles also indicate where he can and cannot go.

There's also some green tiles. These act as a portal to another stage.

Both of these types of tiles work in exactly the same way:

  1. The little dude attempts to move onto a tile on the stage.
  2. The game calculates his future position and checks if there are any special tiles in that position.
  3. If there's a red tile, we stop his movement.
  4. If there's a green tile, we "activate the portal" and load the next stage.

This is a very basic implementation of collision detection. If we wanna get fancy, we could (and eventually will) introduce collision based on the direction the little dude is facing. This is how we'll implement interacting with the environment in the next area.
